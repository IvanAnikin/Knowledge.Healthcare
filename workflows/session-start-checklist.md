# Session Start Checklist

Use this checklist at the beginning of every work session to establish context before acting.

## Checklist

- [ ] **Read `AGENTS.md`** -- Understand the rules and expectations for working in this knowledge layer.
- [ ] **Read `current-priorities.md`** -- Know what is active, blocked, and recently completed.
- [ ] **Read the relevant application overview** -- If working on a specific application, read its `applications/app-*-overview.md`.
- [ ] **Read the latest handoff** -- Check `handoffs/` for the most recent entry to understand what the last session accomplished and what it recommends.
- [ ] **Identify unknowns** -- Before starting work, list what you do not know and what you need to find out.
- [ ] **State assumptions** -- If you are proceeding based on assumptions, write them down so they can be verified.

## Notes

- If any of the files above are empty or contain only placeholders, note that as a gap and consider whether filling them in is part of the current session's goal.
- If the current session's goal contradicts information in these files, flag the conflict before proceeding.
