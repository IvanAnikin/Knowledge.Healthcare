## Problem

After ingest and mapping, the app still needs deterministic CGM-specific validation before analysis can be trusted. The system must handle LibreView record types, normalize glucose units to mmol/L, and surface structured warnings/errors in the UI.

## Desired Outcome

- CGM schema validation is implemented.
- LibreView record type handling is implemented for types `0`, `1`, and `6`, with warnings for unknown types.
- Glucose values are normalized to mmol/L.
- Structured validation warnings/errors are returned by the backend and rendered in the frontend UI.

## Scope

- CGM schema validation.
- LibreView record-type handling.
- mmol/L normalization.
- Structured warnings/errors UI.

## Constraints

- Build on the upload/preview/mapping flow from TASK-0022.
- Keep validation deterministic and separate from analytics.
- Errors must block analysis; warnings must not.
- Respect the immediate deletion rule for uploaded source files after parsing/validation and analysis handoff.

## Validation

- A valid LibreView-style sample passes validation.
- Invalid mappings or malformed CGM inputs produce structured blocking errors.
- Type `6` records are excluded correctly.
- Unknown record types generate warnings.
- Unit normalization to mmol/L is verifiable on sample data.
