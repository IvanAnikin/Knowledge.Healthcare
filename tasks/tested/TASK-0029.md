## Problem

The current `Health-Analysis` web UI is functionally usable for milestone testing, but it still reads like a plain internal utility rather than a polished clinician-facing product.

The main upload page and analysis page currently rely on local inline styles, white cards, basic tables, and a minimal page shell. That makes the workflow understandable only with effort, reduces visual hierarchy, and does not yet deliver the modern, comfortable, high-trust product feel the user wants.

The user now wants this captured as a concrete UI/UX improvement task with a clear visual direction inspired by the `Knowledge.Healthcare` dashboard: dark mode, glass-like surfaces, rounded corners, soft background lights, and subtle animation, while still keeping the app minimal, fast, readable, and clinically comfortable.

## Desired Outcome

- `Health-Analysis` gets a coherent visual system and layout language across the current upload, preview, mapping, validation, analysis, chart, and LLM-summary surfaces.
- The UI feels modern, minimalistic, user-friendly, fast, and comfortable rather than utilitarian.
- The redesign is inspired by the `Knowledge.Healthcare` dashboard aesthetics without becoming a literal clone.
- The core workflow becomes easier to scan and navigate, with stronger hierarchy, clearer action states, and better responsiveness on desktop and mobile.
- The redesign preserves trust, readability, and deterministic-first clinical framing.

## Requirements

- Add a reusable app-4 visual system for colors, spacing, radii, shadows, borders, typography, and surface treatments instead of relying on per-file inline styling.
- Support dark mode with the dashboard-inspired look as the primary visual direction, and include a persistent theme toggle if light mode is retained.
- Introduce glass-style panels and rounded surfaces for the page shell and major content cards while maintaining strong text contrast and chart/table readability.
- Add restrained ambient background-light effects to the main shell so the interface feels premium without becoming noisy or distracting.
- Rework the main upload flow so the user can clearly understand the current step, available next action, and validation state at a glance.
- Improve the validation experience so errors, warnings, success state, severity, and suggested fixes are more scannable and less raw.
- Rework the analysis page into a clearer summary-first hierarchy before detailed tables and charts.
- Keep data-dense content comfortable by improving spacing, grouping, headings, table treatment, and chart containers.
- Preserve clear presentation of clinician disclaimers, heuristic labels, and data-quality warnings.
- Ensure the main surfaces work comfortably on both desktop and mobile widths.
- Keep interactions fast-feeling; avoid heavy effects, oversized transitions, or layouts that make charts/tables harder to use.
- Respect reduced-motion preferences and accessibility basics for focus states, contrast, and keyboard usability.

## Acceptance Criteria

- The app has a consistent app-wide visual language across `/`, `/analysis/[id]`, chart panels, stats tables, alerts, and disclaimers rather than isolated one-off inline card styles.
- A dark-mode presentation inspired by `Knowledge.Healthcare/dashboards/task-dashboard.html` is implemented and visually coherent across the main app-4 workflow.
- If both themes are supported, the chosen theme persists across reloads.
- The home workflow presents upload, preview, mapping, validation, and run-analysis steps with clearer hierarchy and obvious primary actions on both mobile and desktop.
- Validation errors and warnings are visibly distinct, easy to scan, and include enough context for the user to know what to fix next.
- The analysis page surfaces the most important summary information first and keeps tables/charts readable without feeling cramped or visually flat.
- Ambient background lights and glass effects remain subtle enough that text, charts, and tables stay readable and comfortable for prolonged use.
- Motion is present only as restrained enhancement, and reduced-motion preference disables or minimizes non-essential animation.
- Existing app functionality does not regress: upload, preview, mapping submission, validation display, analysis start, analysis rendering, and LLM summary actions still work.
- Relevant frontend tests are updated or added for any newly introduced theme/state behavior, and the standard app validation/build checks pass.
