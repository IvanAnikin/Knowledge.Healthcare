## Problem

The application needs a first end-to-end data-ingest path before any analytics can run. Users must be able to upload CSV/XLSX files, preview the parsed rows, and submit a dataset type plus manual column mapping.

## Desired Outcome

- CSV and XLSX loading works in the backend.
- The backend exposes upload, preview, and mapping endpoints.
- The frontend provides a basic upload and mapping UI.
- A user can upload a sample dataset, preview rows/columns, and submit mapping information successfully.

## Scope

- CSV/XLSX loaders.
- Upload endpoint.
- Preview endpoint.
- Mapping endpoint.
- Basic upload/mapping UI.

## Constraints

- Build on the monorepo scaffolding from TASK-0021.
- Keep the slice focused on ingest/preview/mapping only; validation rules and analytics belong to later tasks.
- Preserve the documented API shape under `/api/v1` where practical.
- Use the synthetic CGM sample to drive the happy path.

## Validation

- Sample CSV upload succeeds.
- Preview endpoint returns parsed rows and detected columns.
- Mapping submission succeeds and persists the chosen dataset type + column mapping.
- Frontend flow reaches upload -> preview -> mapping submission without manual backend intervention.
