## Problem

The validated input flow still needs the actual deterministic CGM analytics engine. Without this, the app cannot produce the statistical outputs, trend segments, episode detection, or heuristic post-meal analysis that define the MVP.

## Desired Outcome

- The CGM analytics module computes:
  - time in range
  - mean / median / SD / CV / GMI
  - dayparts
  - monthly backward windows
  - episodes
  - rate-of-change
  - post-meal heuristic
- The analytics module is versioned and testable.
- Core CGM numeric outputs are reproducible on repeated runs with the same input.

## Scope

- Time in range.
- Mean / median / SD / CV / GMI.
- Dayparts.
- Monthly backward windows.
- Episodes.
- Rate-of-change.
- Post-meal heuristic.

## Constraints

- Build only on validated/normalized input from TASK-0023.
- Keep all analytics deterministic.
- Mark the post-meal analysis as heuristic in outputs and assumptions.
- Add fixture-based backend tests for numeric correctness and reproducibility.

## Validation

- Synthetic/reference datasets verify time-in-range edge cases.
- Variability metrics match expected reference outputs.
- Episode detection handles threshold/duration boundaries correctly.
- Repeated runs with the same input and module version produce identical numeric outputs.

## Notes

- This is the core of Milestone `M2` and the main clinical-value engine for the MVP.
- Do not combine chart rendering or LLM work into this task.
