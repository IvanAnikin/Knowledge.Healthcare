## Problem

The MVP requires an LLM-generated narrative summary, but it must be safely constrained, backed by deterministic outputs, and auditable. The app still needs the provider abstraction, Azure OpenAI adapter, UI rendering of Findings vs. Interpretation, clinician disclaimer, and audit logging for summary calls.

## Desired Outcome

- `LLMProvider` abstraction exists.
- Azure OpenAI adapter exists as the default provider.
- The UI renders Findings vs. Interpretation clearly.
- The clinician disclaimer is visible with every LLM-generated summary.
- Every LLM summary call writes an audit record.

## Scope

- `LLMProvider` abstraction.
- Azure OpenAI adapter.
- Findings vs Interpretation output.
- Clinician disclaimer.
- Audit logging.

## Constraints

- LLM input must be structured computed outputs, not raw patient rows by default.
- Keep Azure OpenAI as the default initial provider while preserving the abstraction.
- Enforce the prompt and output rules from `docs/SAFETY_AND_GOVERNANCE.md`.
- Do not broaden this task into Phase 2/3 sandbox work.

## Validation

- A completed CGM analysis run can generate a summary through Azure OpenAI.
- Output contains Findings and Interpretation sections.
- Output cites concrete computed values.
- The clinician-verification disclaimer is present.
- An audit record exists for the summary call.

## Notes
