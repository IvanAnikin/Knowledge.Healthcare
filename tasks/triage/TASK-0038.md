## Problem

The KH control layer currently has no programmatic retrieval over its own markdown corpus and no observability into LLM usage across agents. OpenCode subagents must manually read growing lists of files per session, leading to context bloat and missing relevant knowledge. There is also no way to track token spend or latency across Azure OpenAI calls made by agents working in this repo.

These two gaps are listed as active blockers in `current-priorities.md`:
1. **"no observability"** — no token/cost/latency tracking for agent LLM calls
2. **"manual growing agent reading list"** — subagents cannot query the corpus; they read files ad hoc

## Desired Outcome

A self-contained Python package `kh_agent/` exists in the KH control layer repo and provides:

1. **FAISS-backed retrieval** — per-app and per-concern indexes over the KH markdown corpus, queryable via `kh-agent retrieve "<query>"` CLI
2. **Token-tracking LLM client** — wraps `anote-openai` (Azure OpenAI), emits structured telemetry (tokens, cost estimate, latency) to a local log
3. **CLI surface** — `kh-agent ingest / retrieve / status / call` for use by OpenCode subagents via shell tool
4. **Agent prompt integration** — AGENTS.md and relevant agent definitions updated to invoke `kh-agent retrieve` as their first action at session start

Iteration 2 items (LLM-as-Judge, workflow engine ports) are explicitly deferred and out of scope for this task.

## Constraints

## Unknowns

The following open questions are carried from spec §13 and must be resolved before or during Phase 1.1:

1. **FAISS vs. ChromaDB** — spec leans FAISS (zero-server, embeddable); confirm no ChromaDB preference
2. **Embedding model** — `text-embedding-3-small` assumed; confirm deployment name on `anote-openai`
3. **Index storage path** — `.kh_agent_index/` at repo root assumed; confirm acceptable (gitignored)
4. **Telemetry sink** — local JSONL file assumed; confirm no external sink (Prometheus, Azure Monitor) required for Phase 1
5. **CLI invocation context** — confirm OpenCode shell tool has access to invoke `kh-agent` in the KH repo working directory
