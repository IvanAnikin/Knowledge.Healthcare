## Implementation Summary

- Root cause: the `/cenik` route rendered `PageHeader` with `dict.pricing.heading` plus `dict.pricingPage.subtitle`, while `Pricing.tsx` also rendered its own internal heading block with the same `dict.pricing.heading` plus `dict.pricing.subheading`. That created two stacked title-style blocks.
- Implementation: removed the inner heading/subheading block from `Pricing.tsx`, so the pricing card now follows the route-level `PageHeader` directly.
- Locale impact: both Czech and English routes were affected because they shared the same component and dictionary structure.

## Validation Evidence

- `npx tsc --noEmit` passed cleanly.
- `npm run build` succeeded and prerendered routes successfully.
- Spacing remained correct after the removal because the route-level `PageHeader` stays in place and the pricing card retains its wrapper spacing.

## Files Changed During Implementation

- `ANOTE-web/src/components/sections/Pricing.tsx` — removed the duplicate inner heading/subheading block that rendered `t.heading` and `t.subheading`.
