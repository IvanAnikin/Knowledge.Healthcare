## Problem

The `id="features"` section on the homepage has two interaction/visual issues:

1. **Anamnéza container animation:** The anamnéza shortcuts currently sit statically inside the window. The desired behavior is for them to float randomly and bounce off each other (particle-like or physics simulation animation).
2. **Workflow diagram labels:** The current diagram shows `Nahrát Stop Kopírovat` as the three workflow steps. This does not communicate the actual end-to-end workflow, which has two possible final outcomes: automatic email dispatch OR manual copy.

## Desired Outcome

### Anamnéza animation
- Anamnéza shortcut chips/tags inside the anamnéza container float and move randomly within the bounding window.
- They bounce off the container walls and off each other (collision detection or simulated).
- The animation is visually smooth and does not affect page performance.
- The text `13 sekcí` is removed from the anamnéza container.
- The text `funguje i bez internetu` is removed from the anamnéza container.
- The text `hlas pacienta nikdy neopustí telefon` is removed from the anamnéza container.

### Workflow diagram
- The diagram steps are updated from `Nahrát Stop Kopírovat` to:
  - Step 1: `Nahrát`
  - Step 2: `Stop`
  - Step 3 (fork): a simple fork arrow branches from `Stop` into two parallel outcomes:
    - `Odeslat e-mailem` (primary branch)
    - `Kopírovat` (secondary branch)
- The fork must be visually simple — a single arrow splitting into two, with both labels clearly readable.
- The design must be mobile-friendly (both branches legible on narrow screens; stacked or inline layout acceptable).

## Constraints

- Framer Motion (v12+) is already available in the project and is the preferred animation library.
- The animation must not cause layout shifts or performance degradation on mobile.
- The diagram update is a visual/copy change within an existing component — do not redesign the entire features section.
- No new npm packages should be introduced without explicit approval. The animation should be achievable with Framer Motion or CSS.

## Rework Validation (2026-04-23)

- **Result:** pass for implementation completion
- **Changed files:**
  - New: `BouncingChips.tsx` — self-contained floating/bouncing chip animation.
  - Modified: `Features.tsx` — imports `BouncingChips`, renames the middle `featurePlaceholders` entry from `templates` to `anamneza`, and renders 5 anamnéza section chips (`NO`, `RA`, `OA`, `FA`, `AA`) in a 160 px tall container. The previous templates visual and existing workflow fork remain intact in the file.
- **Animation details:**
  - `requestAnimationFrame` loop with direct writes to each chip's `style.transform` so there is no React re-render per frame.
  - `ResizeObserver` tracks container dimensions and re-clamps chips into bounds on resize.
  - `IntersectionObserver` pauses the loop when the component is off-screen.
  - `prefers-reduced-motion: reduce` places chips once and does not start the animation loop.
  - Wall collisions use elastic bounce via clamped positions and negated velocity component.
  - Chip-chip collisions use O(n^2) AABB overlap checks (`n=5`, so 10 pairs/frame), swap velocities, and separate positions along the smaller-overlap axis to prevent sticking.
  - Initial velocities are randomized (`0.5–1.1 px/frame`) with staggered initial placement to avoid spawn overlap.
  - Chips use `will-change: transform` and `translate3d(...)` for GPU compositing.
- **Placement rationale:** `TASK-0001` had already reduced `features.items` to 3 entries, removing the original dedicated anamnéza feature card assumed by the spec. The animation was wired into the surviving middle feature slot as the closest analogue so the required floating `NO` / `RA` / `OA` / `FA` / `AA` visual now exists in the features section. The old static templates visual remains in the file but is no longer referenced by `featurePlaceholders`.
- **Validation performed:**
  - `npx tsc --noEmit` — passed.
  - `npm run build` — passed; all 28 static pages generated with no build warnings related to this change.
  - IDE diagnostics on both modified files — clean.
  - Client chunk inspection confirmed `BouncingChips` is bundled into the shared client chunk loaded by `/[lang]` routes.
- **Still requires browser confirmation:**
  - Visual smoothness of the floating/bouncing motion on `/cs#features` and `/en#features`.
  - Chips stay inside the container across viewport widths 320 px → 1440 px.
  - The animation pauses when the section is scrolled off-screen.
  - The `prefers-reduced-motion: reduce` fallback renders all 5 chips visibly without overlap.
- **Sufficiency to move to `tested`:** Not yet. The supplied evidence is sufficient to move from `active` to `done`; browser re-validation is still required before advancing to `tested`.
