## Problem

The homepage features section already includes an animated anamneza shortcuts frame, but it currently shows only five shortcut chips: `NO`, `RA`, `OA`, `FA`, and `AA`.

The requested marketing update is to show a larger set of anamneza shortcuts inside that same animation frame on the main page so the visual better communicates the breadth of structured anamneza capture.

## Desired Outcome

- The homepage features section displays more anamneza shortcut chips in the existing animated frame.
- The implementation keeps the current floating/bouncing behavior intact.
- The expanded chip set remains readable and contained within the visual on desktop and mobile widths.
- The surrounding feature card layout and copy remain stable unless a small supporting adjustment is required.

## Confirmed Facts

- The ANOTE-web homepage features section is implemented in `src/components/sections/Features.tsx`.
- The current feature placeholder mapping is `transcript`, `anamneza`, and `workflow`.
- The anamneza visual renders `BouncingChips` inside a `w-full h-40 p-2` wrapper in `Features.tsx`.
- The implemented chip list is `[`NO`, `RA`, `OA`, `FA`, `AA`, `GA`, `SA`, `EA`]`.
- The animation implementation lives in `src/components/animations/BouncingChips.tsx`.
- `BouncingChips` already accepts a dynamic `chips: string[]` prop, so the chip count is not hardcoded to five at the component boundary.
- The reported implementation changed only `Features.tsx`; `BouncingChips.tsx` was left unchanged.

## Constraints

- Keep scope limited to the homepage features animation in ANOTE-web.
- Preserve the existing animation style unless additional chips make a small spacing, sizing, or container-height adjustment necessary.
- Do not redesign the full features section.
- Validate that the larger chip set does not cause clipping, excessive overlap, or obvious visual regression on narrow screens.
- Do not invent the final shortcut labels if a canonical expanded list is not already documented; use the product-owner-provided list.

## Suspected Affected Areas

- `ANOTE-web/src/components/sections/Features.tsx` - current anamneza visual wrapper and chip list
- `ANOTE-web/src/components/animations/BouncingChips.tsx` - only if chip density or collision behavior needs a small adjustment
- `ANOTE-web/src/dictionaries/cs.json` and `src/dictionaries/en.json` - only if nearby copy needs to mention the broader shortcut set

## Unknowns

- Whether the added `EA` label is the intended canonical shortcut abbreviation for long-term product/marketing use.
- Whether the denser 8-chip motion feels visually comfortable on a narrow mobile viewport under real browser rendering.
