## Problem

`Bezpečnost` currently exists as an embedded section inside the report-types page instead of as its own page. This is a content-architecture and routing bug, not a net-new feature.

The current implementation in `src/app/[lang]/typy-zprav/page.tsx` renders the `Privacy` section inline inside a wrapper with `id="security"`, and the homepage trust-strip link points to `/typy-zprav#security` (or `/en/report-types#security`).

## Desired Outcome

- `Bezpečnost` exists as its own dedicated page rather than being embedded in `Typy zpráv`.
- The current security content is removed from the report-types page.
- The homepage `Více o bezpečnosti` link points to the dedicated security page instead of the report-types anchor.
- Localized routing, metadata, and sitemap entries are updated accordingly.

## Confirmed Facts

- `src/app/[lang]/typy-zprav/page.tsx:15-17` imports `Privacy` and renders it inside the report-types page.
- `src/app/[lang]/typy-zprav/page.tsx:68-70` wraps that content in `<div id="security">`.
- `src/components/sections/TrustStrip.tsx:18` defines the destination slug as `typy-zprav` / `report-types`.
- `src/components/sections/TrustStrip.tsx:45-49` links `Více o bezpečnosti` to `${prefix}/${reportTypesSlug}#security`.
- The requirement pack explicitly says `Bezpečnost` should be a separate page and moved out of `Typy zpráv`.
- The requirement pack explicitly says this should be treated as a bugfix / content architecture correction, not a brand new feature.

## Constraints
