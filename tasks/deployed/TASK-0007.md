## Problem

The homepage displays a container labelled `Používají lékaři v České republice`, rendered by the `LogoBar` component. This container should be removed entirely from the main page.

## Desired Outcome

- The `LogoBar` component is no longer rendered on the homepage.
- No broken layout or spacing artefacts remain after removal.
- Orphaned dictionary keys (`logoBar.*`) are removed from both `cs.json` and `en.json`.

## Confirmed Facts (post-investigation)

- **Component:** `src/components/sections/LogoBar.tsx` — confirmed as the sole renderer of `Používají lékaři v České republice`.
  - The string is at `dict.logoBar.label`, accessed at `LogoBar.tsx:27` as `{t.label}`.
  - The component also renders `dict.logoBar.logos` (array of hospital names including `"Vaše klinika?"`).
- **Call site:** `src/app/[lang]/page.tsx` — imported at line 5, rendered at line 42 as `<LogoBar dict={dict} />`, immediately after `<Hero>` and before `<HowItWorks>`.
- **Dictionary keys:**
  - `cs.json` line 33: `"label": "Používají lékaři v České republice"` — key path `logoBar.label`
  - `en.json` line 33: `"label": "Used by doctors across the Czech Republic"` — same key path
  - Full `logoBar` object (label + logos array) exists in both files and should be removed.
- **TrustStrip is NOT involved.** `TrustStrip.tsx` renders `dict.trustStrip.heading` and `dict.trustStrip.points` — unrelated to this task.
- **No other pages or components reference `LogoBar`** — safe to remove the import and component file.

## Constraints

- Do not touch `TrustStrip`.
- Remove the full `logoBar` key object from both dictionaries (not just the `label` sub-key).
- Verify no spacing gap is left in the homepage layout after removal.
