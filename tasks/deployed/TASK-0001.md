## Problem

The marketing site contains several misleading claims about how ANOTE handles audio and connectivity:

1. Phrases like "Funguje offline" appear throughout the site. This is not universally true — report generation always requires a remote model. Only local transcription modes work fully offline.
2. Phrases like "Hlas pacienta nikdy neopustí telefon" are presented as universal behavior. This is only true for local transcription modes, not for the cloud transcription mode.
3. The distinction between cloud and local transcription modes is not clearly communicated anywhere. The site implies a single mode of operation.
4. References to the transcription technology (Whisper vs. gpt-4o-transcribe) are inconsistent or misleading. Cloud mode uses `gpt-4o-transcribe`; local mode uses Whisper.

The FAQ currently contains "Je nahrávka pacienta někam odesílána? Audio zůstává na zařízení." — this is incorrect for cloud transcription mode and must be removed.

## Desired Outcome

- All occurrences of "Funguje offline" are removed from the site.
- All occurrences of "Hlas pacienta nikdy neopustí telefon" are removed or scoped accurately (only stated when describing local mode specifically).
- The site clearly communicates that ANOTE supports multiple transcription modes:
  - Local modes (small to large Whisper models): audio stays on device, works offline for transcription
  - Cloud mode (`gpt-4o-transcribe`): better results, faster, but audio leaves the device
- FAQ item "Je nahrávka pacienta někam odesílána? Audio zůstává na zařízení." is removed.
- FAQ item "Mohu ANOTE napojit na svůj nemocniční systém?" is replaced with a positive answer explaining that transcription can run offline and report generation can happen later when the phone is online. The implementing agent should draft the final Czech copy during implementation, following the copy guidelines in the **Copy Guidelines** section below.
- FAQ item "Jaké typy vyšetření aplikace podporuje?" is updated to mention custom report types: "uděláme vám typ na míru".
- Whisper and gpt-4o-transcribe are mentioned accurately without over-repetition: cloud = gpt-4o-transcribe, local = Whisper.
- The certification disclaimer `ANOTE není certifikovaný zdravotnický prostředek. Slouží výhradně jako nástroj pro podporu lékařské dokumentace.` is removed from all locations where it currently appears.

## Copy Guidelines

The implementing agent must draft all new Czech copy during implementation (no external copy approval is required). Apply the following rules to every new or replaced string:

1. **Be concise.** Prefer one sentence over two. Avoid marketing superlatives.
2. **Be product-facing.** Write for a doctor evaluating a tool, not a general consumer.
3. **No absolute claims.** Do not say "vždy", "nikdy", "veškerá data zůstávají na zařízení" unless the claim is unconditionally true for all modes.
4. **Clearly distinguish modes.** When describing transcription, always make it clear whether the statement applies to local mode, cloud mode, or both. Suggested framing:
   - Local: "Lokální přepis probíhá přímo na telefonu — nahrávka neopouští zařízení."
   - Cloud: "Cloudový přepis využívá `gpt-4o-transcribe` — rychlejší a přesnější, nahrávka je odeslána ke zpracování."
   - Both: "ANOTE podporuje lokální i cloudový přepis. Výsledky zpráv jsou vždy generovány online."
5. **Transcription technology.** Use `gpt-4o-transcribe` for cloud, Whisper (bez dalšího přívlastku) for local. Do not over-repeat model names — mention each model once per section at most.
6. **Avoid "offline" as a blanket claim.** If connectivity must be mentioned, be specific: "přepis funguje offline" (for local mode only), never "aplikace funguje offline" without qualification.

The agent should produce proposed Czech strings alongside any code changes and note them clearly in the commit or PR description for review.
