## Problem

The dedicated security page currently stacks two heading areas for the same content.

The first heading/subtitle is rendered correctly on the white `PageHeader` area. Immediately below, the dark security section renders another heading/subheading under the shield icon, causing the title treatment to feel duplicated. The user wants the first white-background heading kept as-is and the second one removed.

The same page also has too much vertical padding above the shield icon. The requested change is to make that top spacing about half of its current distance.

## Desired Outcome

- `/bezpecnost` keeps the existing white-background `PageHeader` title/subtitle unchanged.
- The duplicate heading/subheading beneath the shield icon is removed.
- The security cards and CTA section remain intact.
- The top padding above the shield icon is reduced to roughly half of its current value without making the section feel cramped.
- The same fix applies cleanly to `/en/security`, since both locales share the same page/component structure.

## Confirmed Facts

- `src/app/[lang]/bezpecnost/page.tsx` renders `<PageHeader title={t.title.replace(" — ANOTE", "")} subtitle={t.subtitle} />` followed by `<Privacy dict={dict} />`.
- `src/components/sections/Privacy.tsx` renders the shield icon plus a second heading block from `dict.privacy.heading` and `dict.privacy.subheading`.
- `Privacy.tsx` is only referenced from `src/app/[lang]/bezpecnost/page.tsx` in the current repo.
- The current section spacing in `Privacy.tsx` is driven by `className="py-24 sm:py-32 ..."`, which is the likely source of the excess space above the shield.
- Czech dictionary copy currently includes both `securityPage.subtitle` (`"Navrženo pro evropské zdravotnictví"`) and `privacy.subheading` (`"Navrženo pro evropské zdravotnictví"`), which is why the subtitle appears duplicated in practice.
- English uses the same shared structure, with matching `securityPage` and `privacy` copy patterns.

## Constraints

- Keep the fix narrow and localized to the security page experience.
- Preserve the first white-background header exactly as requested.
- Do not redesign the privacy/security cards section.
- Prefer the smallest correct implementation; avoid unnecessary dictionary churn if the duplicate block can be removed structurally.
- Verify that the spacing adjustment still looks correct on both desktop and mobile widths.
