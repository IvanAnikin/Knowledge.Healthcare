## What Was Learned

- **Stack reality vs. control-layer notes.** `Health-Analysis/web` runs Next.js **14.2.15** App Router with React 18.3 and Plotly via `plotly.js-dist-min` (`web/package.json:16-37`). No i18n library is installed. The control-layer note "Next.js 16" applies to ANOTE-web (app-3), not app-4.
- **Default UI language is English**, hard-coded at `web/app/layout.tsx:21` (`<html lang="en">`); all UI strings are inline in TSX.
- **Theme persistence is the precedent for locale persistence.** `localStorage["ha-theme"]` plus an inline pre-paint script that sets `data-theme` before hydration (`web/app/layout.tsx:13`). The same pattern is reused in TASK-0032 for `ha-locale` and `<html lang>`.
- **"Episodes" disambiguated** to deterministic CGM hypo/hyper episode-detection output: `Episode` type at `web/lib/api.ts:159-167`, rendered by `EpisodesTable` at `web/components/StatsTables.tsx:258-314`, mounted at `web/app/analysis/[id]/page.tsx:268`. Not the post-meal or recent-meals tables added by TASK-0030.
- **Analysis page is a single Client Component** that does `Promise.all([getAnalysis, getSummary])` and only paints once both resolve (`page.tsx:36-57, 99-101`). All sections render synchronously and `ChartPanel` is statically imported, dragging Plotly into the initial chunk — the perf opportunity TASK-0031 targets.
- **Mapping suggestion is purely client-side** today (`web/app/page.tsx:59-66`, seeding at `:188-192`). The backend stores whatever the client sends; suggestions in `CGM_LOGICAL_FIELDS` (`page.tsx:27-44`) only match English variants — Broz Czech headers fall through to `(none)`.
- **"Broz" = Dr. Broz**, a doctor who forwards CGM CSVs. Verified by reading `samples/broz/Test1.csv:1` and `TEst.csv:1`: 19 Czech-locale headers; first data row shows device `FreeStyle LibreLink` — so Broz CSVs are LibreView Czech-locale exports. Glucose values use comma decimals (`"4,7"`), aligning naturally with `cs-CZ` `Intl` formatting.
- **Existing Broz detection precedent.** `api/app/ingest/loaders.py:83-129` already content-detects the Czech LibreView per-line outer-quoting variant via `_maybe_unwrap_double_quoted_lines` — natural attachment point for the explicit profile detector added by TASK-0033.
- **CSV header typo corrected.** TASK-0033 description had `Skenovat skenovat glukózu mmol/L`; the verbatim CSV header is `Skenovat glukózu mmol/L`. Specs use the corrected literal.
- **Lifecycle precedent.** Prior `spec'd` tasks (e.g. TASK-0014) stayed in `tasks/triage/`; no `tasks/spec/` folder exists. Three task files left in `tasks/triage/` consistent with precedent.

## Decisions Made

- **Recommended execution order: TASK-0032 → TASK-0033 → TASK-0031.** TASK-0033 depends on TASK-0032's Czech vocabulary; TASK-0031 is independent and runs last so its Playwright snapshot refresh can be batched with TASK-0030's pending refresh.
- **TASK-0032/0033 vocabulary contract.** The i18n `cs` dictionary is the single source of truth for Czech CGM field labels. TASK-0033's CSV→canonical-ID mapping references those dictionary keys; canonical field IDs stay English snake_case.
- **No new i18n dependency.** ~30 LOC in-house implementation under `web/lib/i18n/` instead of `next-intl` — keeps the Free-tier Static Web Apps footprint minimal and matches the existing inline-strings codebase scale.
- **No virtualization library for `EpisodesTable`.** Slice-based windowing (200 initial / 1000 warn) is sufficient for realistic CGM episode counts and avoids adding a dependency.
- **Backend changes for TASK-0033 are additive.** New response fields are optional/backward-compatible; the existing client-side `suggestColumn` heuristic remains as fallback when `source_profile_confidence` is below `high`.
- **LLM summary localization deferred.** TASK-0032 only localizes deterministic UI; the summary panel renders a localized "summary is in source language" note. Avoids a separate LLM cost/quality discussion in this slice.
- **Broz = Dr. Broz, a labeled data-source profile.** Treated as a LibreView Czech-locale export, not a vendor/device signature. Detection lives next to existing LibreView preprocessing in `api/app/ingest/`.
- **CSV header literal corrected** to `Skenovat glukózu mmol/L` (TASK-0033 description had a `Skenovat skenovat …` typo).
