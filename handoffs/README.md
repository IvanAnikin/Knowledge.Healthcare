# Handoff Notes

## Purpose

This folder contains session handoff notes. Every substantial work session -- whether by a human or an agent -- should leave a short handoff note when the session produced meaningful context that a future contributor would need.

Handoff notes are the primary mechanism for **continuity between sessions**. They answer the question: "What does the next person need to know to pick up where I left off?"

## When to Write a Handoff

Write a handoff when:

- You made progress on a goal or task
- You discovered something important (a bug, a constraint, a misunderstanding)
- You made or recommended a decision
- You changed documentation or project structure
- You were unable to complete a task and want to explain why

You do **not** need a handoff for trivial sessions that produced no meaningful context.

## How to Write a Handoff

1. Copy `handoff-template.md` to a new file.
2. Name it `YYYY-MM-DD-brief-description.md` (e.g., `2025-01-15-auth-research.md`).
3. Fill in the relevant sections. Keep it concise.
4. Do not edit or delete previous handoff notes. They are append-only records.

## Reading Handoffs

When starting a new session, read the most recent handoff to understand:
- What was the last session's goal?
- What was accomplished?
- What is unresolved?
- What should happen next?
